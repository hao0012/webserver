### tcp_server

```mermaid
flowchart TD
    subgraph tcp_server [TCP Server]
        direction TB

        %% 阶段1：构造
        subgraph ctor [构造阶段]
            direction TB
            C1[初始化 main_loop_] --> C2[初始化 thread_pool_]
            C2 --> C3[初始化 acceptor_<br/>绑定 socket + channel]
        end

        %% 阶段2：启动
        subgraph start [start 启动阶段]
            direction TB
            S1[thread_pool_.start<br/>创建 eventloop_thread 数组] --> S2[main_loop_.run_in_loop<br/>投递 acceptor_.listen]
            S2 --> S3[main_loop_.loop<br/>阻塞监听事件]
        end

        %% 阶段3：新连接处理（核心逻辑）
        subgraph accept [acceptor::listen 新连接回调]
            direction TB
            A1[accept_socket_.accept<br/>获取 conn_fd + client_addr] --> A2{conn_fd >= 0?}
            A2 -- No --> A3[记录错误日志]
            A2 -- Yes --> A4[thread_pool_.get_next_eventloop<br/>轮询选择子 loop]
            A4 --> A5[next_connection_id<br/>生成唯一 ID]
            A5 --> A6[make_shared&lt;tcp_connection&gt;<br/>创建连接对象]
            A6 --> A7[设置用户回调<br/>connect / message / write_complete]
            A7 --> A8[loop.add_connection<br/>加入 eventloop 连接管理 map]
        end

        %% 流程关系
        ctor ==> start
        S3 -. 触发 .-> A1
    end
```

### eventloop_thread_pool

```mermaid
flowchart TD
    subgraph eventloop_flow [EventLoop 核心流程]
        direction TB

        %% 构造阶段
        subgraph ctor [构造阶段]
            direction TB
            C1[初始化 quit_=false<br/>thread_id_] --> C2[创建 eventfd<br/>wakeup_fd_]
            C2 --> C3[初始化 wakeup_channel_<br/>绑定 wakeup_fd_ 读回调]
            C3 --> C4[wakeup_channel_.enable_reading<br/>注册到 poller]
        end

        %% 主循环阶段
        subgraph loop [loop 主循环]
            direction TB
            L1{quit_ == true?} -- Yes --> L2[退出循环]
            L1 -- No --> L3[poller_.poll<br/>阻塞等待活跃事件]
            L3 --> L4[遍历 active_channels]
            L4 --> L5[channel->handle_event<br/>执行各 channel 回调]
            L5 --> L6[run_pending_callbacks<br/>执行 pending 队列中的回调]
            L6 --> L1
        end

        %% 跨线程任务投递
        subgraph run_in_loop [run_in_loop / queue_in_loop]
            direction TB
            R1{是否在 loop 线程?} -- Yes --> R2[直接执行 cb]
            R1 -- No --> R3[加锁放入 pending_callbacks_]
            R3 --> R4[wakeup<br/>向 eventfd write 唤醒]
        end

        %% 连接管理
        subgraph add_conn [add_connection]
            direction TB
            A1[connections_ 添加连接] --> A2[run_in_loop<br/>投递 establish_connection]
        end

        %% 退出阶段
        subgraph quit [quit 退出]
            direction TB
            Q1[quit_ = true] --> Q2{是否在 loop 线程?}
            Q2 -- No --> Q3[wakeup 唤醒 poller]
            Q2 -- Yes --> Q4[下次循环判断退出]
        end
    end

    %% 关系
    ctor ==> loop
    run_in_loop -.-> L3
    add_conn -.-> run_in_loop
    quit -.-> L1
```

```mermaid
    end
```

## eventloop

```mermaid
flowchart TD
    subgraph eventloop [EventLoop]
        direction TB

        C1[构造: 创建 eventfd<br/>初始化 wakeup_channel] --> L1[loop 进入主循环]

        L1 --> Q1{quit_?}
        Q1 -- Yes --> Q2[退出循环]
        Q1 -- No --> P1[poller_.poll<br/>阻塞等待事件]

        P1 --> A1[遍历 active_channels<br/>执行 channel 回调]
        A1 --> R1[run_pending_callbacks<br/>执行队列中的回调]

        R1 --> L1

        %% 跨线程任务投递
        I1[run_in_loop] --> I2{同一线程?}
        I2 -- Yes --> I3[直接执行]
        I2 -- No --> I4[queue_in_loop<br/>加入 pending 队列]
        I4 --> I5[wakeup<br/>写入 eventfd 唤醒]

        %% 连接管理
        AD1[add_connection] --> AD2[connections_ 注册]
        AD2 --> AD3[run_in_loop<br/>establish_connection]

        %% 退出
        Q3[quit] --> Q4[quit_ = true]
        Q4 --> Q5{同一线程?}
        Q5 -- No --> I5
    end
```

## 问题

### 客户端发送了数据，服务端建立连接成功，但是没有处理数据，同一个事件不断就绪

不同事件的 “处理完成” 标准：
1. 读事件 EPOLLIN：处理完成 = 把缓冲区数据全部读完，读完后，内核缓冲区空了 → 不再就绪，LT 模式（默认模式）下就不会再触发了
2. 写事件 EPOLLOUT：只要发送缓冲区不满，就一直就绪！一旦监听 EPOLLOUT，epoll_wait 会立刻、连续、疯狂返回，必须写完数据后，立刻删除 EPOLLOUT 监听
